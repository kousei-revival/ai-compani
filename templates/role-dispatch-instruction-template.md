# 役職への約束付き依頼（リーダー発行）— ひな形

> **使い方**
>
> 1. 下の `（ここに…）` を全部埋める（抜けがあると合意とみなされません）。
> 2. ファイルを保存する（例: `~/ai-company/members/leader/projects/draft-dispatch-2026-04-22.md`）。
> 3. ターミナルで配信:  
>    `~/ai-company/scripts/dispatch-instruction.sh` にこのファイルを渡し、**宛先の役フォルダ名**を並べる。  
>    配信先の AI は、**自フォルダをルートにして** `cd ~/ai-company/members/<役職名>` から会話を開き、**`inbox/` に着いた `INSTRUCTION-` で始まる最新**を最初に読む。

---

## メタ（追跡用）

| 項目 | 内容 |
|------|------|
| **依頼ID** | （例: `LEAD-2026-04-22-01`。リーダーが一意に付ける） |
| **発行日** | YYYY-MM-DD |
| **回答期限** | 日付または「営業日 N 日以内」 |
| **宛先（役職フォルダ名）** | 例: `sns` / `writer` / `researcher`（`members/` 直下の名前） |
| **依頼の種類** | 例: 分析 / 執筆 / デザイン / 事実調査 / その他（一言） |

## 4. 返却形式（必須）

- **置き場所:** 原則 **`~/ai-company/members/leader/inbox/replies/<依頼ID>-<役職名>.md`** に保存（リーダーが集約する）
- **Slack にも出す場合（任意）:** 先頭行を **`【役職フォルダ名】`** で始め、**依頼ID** を1行目か2行目に書く。チャンネルは依頼で指定。規約例は **`~/ai-company/config/slack-roles.md`**。
- **本文に含めること:**
  1. 要約（5 行以内）
  2. 根拠・出典
  3. 不確かな点 / 要確認
  4. 次アクション案（リーダーが役職者会議に載せる前提）

## 5. リーダーが行う次の一歩（合意）

- 返却を受け取ったら、**`meetings/handoffs/YYYY-MM-DD.md` または** `members/leader/projects/` の日次集約に**転記**し、経営者に上げるか判断する。
