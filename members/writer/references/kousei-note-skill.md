# こうせい版 note記事執筆

> **ai-company のライター部屋で使うとき:** スタイルガイドのパスは `references/mikami-writing-style-guide.md` を読む（元の `references/mikami-writing-style-guide.md` と同一内容のコピー）。下書きの保存先は本書「保存ルール」の **`projects/<案件名>/`** に従う。

## 目的
`みかみの文体リズム` を借りながら、`こうせいの専門性` である運動習慣・理学療法・行動変容をテーマにした note 記事を一発で下書き化する。

これは `みかみ本人のコピー` ではない。

- 構成とリズムは借りる
- 題材と体験はこうせいに置き換える
- アドネス哲学は基本的に入れない
- 読後感は「責められた」ではなく「少し動けそう」にする

## 使う時

以下の依頼で使う。

- `こうせい向けにnote書いて`
- `運動習慣テーマで1本書いて`
- `みかみ寄せで、でも中身はこうせいで`
- `理学療法士の視点で習慣化noteを書いて`

## 必須の前提

### 文体の借り方
ベースは `references/mikami-writing-style-guide.md` の以下だけを借りる。

- 冒頭の引き込み
- 短文リズム
- 問いかけ → 即答え
- 1文ごとの改行
- 極短文パンチ
- セクションごとの感情の波
- `$` による太字指示
- `P.S.` で人間味を出す締め

### 借りすぎ禁止
以下はそのまま持ち込まない。

- アドネス哲学
- AIスタートアップ文脈
- 過剰に煽る起業家テンション
- 三上功太の経歴・固有名詞
- `みかみです`
- `僕` 表記

## こうせいの固定設定

- 名乗り: `どうも！理学療法士のこうせいです。`
- 一人称: `ぼく`
- 主戦場: 回復期リハビリテーション病棟
- 専門領域: 運動習慣化の支援、行動変容、続けられない人の心理理解
- 実体験: 病棟で「退院後に運動が続かない」患者を何人も見てきた
- 体現エピソード: トライアスロン競技者として自分でも習慣化を実践
- 核心メッセージ: `運動習慣は意志力ではなく仕組みで決まる`
- 読者: 30〜40代の忙しいビジネスパーソン、自分を責めがちな真面目な人
- CTA: `https://x.com/kousei_habit`

## 記事の基本設計

### 標準アーク
以下の順で組む。

1. 病棟や現場で見たリアルな場面
2. 読者が自分を責める構造の提示
3. `なぜか。` で本質を断言
4. 意志力モデルを崩す
5. 実体験か臨床経験で裏づける
6. 明日から使える小さな設計に落とす
7. `あなたのせいじゃない` で締める
8. `P.S. / P.P.S. / P.P.P.S.` を付ける

### 使いやすい記事タイプ
- `philosophy × experience-review`
- `experience-review`

迷ったら `philosophy × experience-review` にする。

## 執筆ルール

### 冒頭
- `どうも！理学療法士のこうせいです。` で始める
- 最初の10〜20行で病棟や日常のリアルな場面を置く
- `この記事では〜` から始めない
- 3段落以内に核心を置く

### 本文
- 一文は短く
- 1〜3文ごとに改行
- 4文以上の説明が続いたら短いパンチを入れる
- `なぜか。` → 空行 → `$ 結論` の形を最低1回は使う
- 箇条書きより、短文の連打を優先
- 専門用語を使うなら直後に平易な補足を入れる

### こうせいらしさ
- 優しさを残す
- 読者を責めない
- でも結論は曖昧にしない
- `ぼくは現場でこう見てきた` という報告で語る
- `頑張れ` より `戻れる形を作ろう` に寄せる

### みかみリズムの移植ポイント
- 短文3連打
- `でも。`
- `なぜか。`
- `$` 付きの一撃
- 極短文パンチ
- 真面目3段落の後に軽い脱力

### 笑いの温度
- `笑` は使っていい
- ただし自虐・人間味の範囲に留める
- 患者や読者を笑いの対象にしない

## 使うべき具体素材

優先的に入れる。

- 入院中はできるのに退院後に止まる人
- `やらなきゃ` と分かっているのに続かない人
- 忙しい、疲れる、スマホを見る、ソファに座る、で終わる日常
- トライアスロンでも毎日やる気があるわけではない話
- 靴を出す、時間を決める、歯磨きの後にやる、のような小さな設計

## こういう結論に着地させる

よく使う着地点:

- `$ 続かないのは、あなたの根性が足りないからじゃない。続く設計になっていないからです。`
- `$ 運動習慣は、意志力で決まるものじゃない。仕組みで決まる。`
- `$ 習慣化の本質は、自分を律することじゃない。自分が動きやすい形を作ることです。`

## 保存ルール

- **保存先（ai-company）:** `projects/<案件名>/`（例: `projects/2025-habit-note/`）
- **保存先（元の note 記事プロジェクト単体で動かす場合）:** `articles/drafts/` のままでよい
- ファイル名（ai-company 推奨）: `article_article_<slug>_YYYY_MM_DD_YYYY-MM-DD.md` または `note.md`（案件内でわかれば可）
- frontmatter は以下を基本形にする

```yaml
---
title: [タイトル]
type: philosophy × experience-review
status: draft
created: [YYYY-MM-DD]
tags: [運動習慣, 習慣化, 理学療法, 行動変容, 健康]
target_readers: 30〜40代の忙しいビジネスパーソン、運動しなきゃと分かっているのに続かない人、自分を責めがちな真面目な人
estimated_read_time: 8分
ai_disclosure: "AIと一緒に書いた"
---
```

## 最低限の品質チェック

- [ ] 名乗りが `こうせい` になっている
- [ ] 一人称が `ぼく`
- [ ] `$` の強調行が各セクションにある
- [ ] みかみの固有文脈が混入していない
- [ ] 病棟かトライアスロンの具体エピソードがある
- [ ] 読者を責めていない
- [ ] `P.S.` がある
- [ ] 結論が `仕組み` に着地している

## 例

### 例1
入力:
`みかみ寄せで、こうせいの運動習慣noteを最初から1本書いて`

出力方針:
- 体験レビュー × 哲学型にする
- 病棟のリアルから始める
- `続かないのは性格のせいじゃない` を前半で断言
- 後半で小さな設計に落とす

### 例2
入力:
`運動習慣は意志力じゃない、でnote本文作って`

出力方針:
- タイトルはそのまま使う
- `なぜか。` を使って論点を切る
- トライアスロンの自分の話を挟む
- 最後は `あなたのせいじゃない` で閉じる
