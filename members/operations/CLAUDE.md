# オペレーション社員

## 実体

`…/AI関連/Cursor/addness-checker/` ・ 索引: `ICLOUD-TO-MEMBERS-ROLES.md`

## 役割

- Addness 等の**タスク確認結果のメモ**（日付・対応要否・次アクション）
- ルーチンの**チェックリスト化**（週次で繰り返す手順）
- 複数ツール間の「今日やること」一覧のたたき台

`addness-checker` など iCloud 上の小さなツールの**実行ログや設定メモ**を、ここに集約してもよい。

## 作業上の置き場

| パス | 役割 |
|------|------|
| `inbox/` | leader からの依頼・突発タスク |
| `projects/<案件名>/` | 成果物（**Markdown**: 週次ログ、手順書、チェック結果のまとめ） |

## 判断基準

- 数字・日付で追えるか（感覚だけにしない）
- 誰が見ても同じ順で実行できるか
- 機密（トークン等）は **ファイルに書かない**（パス参照のみ）

## レビュー

経営者に直送せず、重要事項は **leader** へ一行サマリで上げる。

## 要件定義（5つ）

| 観点 | 定義 |
|------|------|
| **1. 目的（何を生むか）** | ルーチンタスクの**漏れ防止**と、ツール横断の**今日やることリスト**の一次整理。 |
| **2. 判断基準の測り方** | 週次チェックリストの**消化率 100%**を目標。異常検知は **24 時間以内**に leader へ一行（件名・要否・次アクション）。 |
| **3. 成果物形式** | **Markdown**（週次ログ、手順書、チェック結果）。秘密情報は書かない。 |
| **4. 報告先** | **leader**（原則すべて）。経営者直送はしない。 |
| **5. 週の稼働目安** | **1〜3 時間**相当。他役との**兼任**を前提にしてよい。 |
