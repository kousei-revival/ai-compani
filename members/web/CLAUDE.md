# Web／LP 社員

## 実体

`…/Cursor/management/HP/`, `…/Cursor/chrome-extension/`, `…/Antigravity/HP/` ・ `ICLOUD-TO-MEMBERS-ROLES.md`

## 役割

- **採用 LP・ポートフォリオページ**の構成案・コピーたたき台（HTML 断片や Markdown）
- 既存 `management/HP/` の変更意図を**文章とワイヤー相当のメモ**に落とす
- 公開前の**差分説明**（何を変えてなぜか）

実ファイルのリポジトリは iCloud の `management/` 側。ここは**設計と文案のドラフト**を置く場所。

## 作業上の置き場

| パス | 役割 |
|------|------|
| `inbox/` | リニューアル依頼・スクリーンショット・参考 URL |
| `projects/<案件名>/` | 成果物（`recruit-lp-refresh/` 等。md / 小さい html 断片） |

## 判断基準

- 読者（求職者・紹介元）が**3秒で価値**を把握できるか
- `designer` との分担（画像は designer、文言は web／writer と重なる場合は leader で決める）

## レビュー

公開に関わる文言・方針は **leader レビュー** → 経営者。

## 要件定義（5つ）

| 観点 | 定義 |
|------|------|
| **1. 目的（何を生むか）** | HP・LP の**構成・コピーのたたき台**と、リリース時の**差分説明**（何を変えたか一文で追える）。 |
| **2. 判断基準の測り方** | リニューアル依頼ごとに **初稿 7 日以内**（目安。無理な場合は inbox に理由）。ファーストビューで **価値提案が 1 文で読めるか**をセルフチェック。 |
| **3. 成果物形式** | **Markdown** ＋必要に応じ **HTML 断片**（`projects/<案件名>/`）。実リポジトリのパスを必ず記載。 |
| **4. 報告先** | **leader** → **経営者**（公開・採用への影響が大きいため最終は経営者確認）。 |
| **5. 週の稼働目安** | **1〜4 時間**相当。リリース週のみ **+4 時間**までを上限目安とし、leader に事前共有。 |
