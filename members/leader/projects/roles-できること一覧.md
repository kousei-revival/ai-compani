# 役割ごと「何ができるか」（ai-company 社員の整理）

`members/<役>/CLAUDE.md` を要約。**実際の作業**は `inbox` 依頼＋`projects` 成果物。重なるときは **leader が文脈を振り分け**。

---

| 役（フォルダ名） | 主にやること（できること） |
|------------------|----------------------------|
| **leader** | 全員のタスク集約・**レビュー**（承認/差し戻し）・**経営者向け手渡し**（`meetings/handoffs`）・dispatch で各役 `inbox` へ流す・X 回りの監督サマリ・**営業4役の週次の矢印**（sns→researcher→cs）の指示。 |
| **sns** | **X 等の投稿設計**（枠・時間帯）・**投稿文案**・Buffer/シートの手順とログ・**X 自動投稿の運用オーナー**（`RUNBOOK`、トピック方針）。**認知**（営業パイプの入口）。 |
| **writer** | **ブログ・メルマガ・LP コピー**・SNS 短文、**note / X 長文**（リズム・`references/` の手順）・`voice-reference` に合わせた文面。 |
| **designer** | **バナー・LP ビジュアル**・AI 画像＋従来ツール・**X 投稿用画像**（`RUNBOOK` / sns 依頼に連動、サイズ・トーン）。 |
| **researcher** | **クラウドソーシング等の案件探索**（`job-scout`、スプシ連携）・**候補の優先度たたき台**・応募用の**素材整理**（**送信は人**）。**新規の入口**。 |
| **web** | **採用 LP / ポートフォリオ**の**構成＋コピーたたき台**（HTML 断片・md）・**公開前の差分説明**（実ファイルは多く iCloud `management/HP` 側）。 |
| **video** | **台本・構成案**（秒数・テロップ）・撮影/編集の**チェックリスト**・1本ごと `projects/<案件名>/`。 |
| **operations** | **ルーチン確認**・Addness 等の**タスク確認メモ**・**チェックリスト**・週次で同じ手順を回すための**手順メモ**集約。 |
| **customer_success** | **契約後の伴走**・ロードマップ案・**日報/Todo たたき台**・**週次振り返り**（数字）・`projects/<クライアントコード>/` に分ける。**受注あとの続き**が主戦場。 |
| **academic** | 症例・**学術班**の**スケジュール案・スライド構成**・**ガイド文書の下書き**・発表/ロードマップの**更新ログ**（iCloud 学術班と連携）。 |

---

## 営業パイプにだけ一言（4 役）

**認知（sns） → 機会（researcher） → 受注後（customer_success）**。  
`leader` が「今週どこを回すか」タグ（例: 探索のみ / 発信強化）を付けると迷いにくい（会社共通 `CLAUDE.md` の定義に合わせる）。

---

## 「いま何してるか」はどう見るか

1. 各 `members/<役>/inbox/` … **来ている依頼**（`INSTRUCTION-*`）  
2. 各 `members/<役>/projects/` … **出ている成果物**  
3. `meetings/handoffs/当日.md` … **全体のその日の合意**（あれば）  

正の役割定義の全文は、いつも通り各 **`members/<役>/CLAUDE.md`**。
