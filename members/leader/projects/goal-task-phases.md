# 目標・フェーズ・タスク（リーダー用たたき台）

`meetings/handoffs/` の補助として、**北極星 → フェーズ → やること**を並べる。数字は週次で見直してよい。

---

## 北極星（目標）— 3 本

| # | 目標 | わかるようになったら |
|---|------|----------------------|
| G1 | **経営者が短時間で判断できる**（数字・手渡し・リスクが一箇所に） | `meetings/handoffs/当日.md` が迷わず読める |
| G2 | **各役（AI 社員）の依頼が止まらない**（出す・戻すがルール化） | `inbox/INSTRUCTION-*` → `replies/` の往復が週1以上動く |
| G3 | **外に向けた発信と案件がパイプでつながる**（営業・SNS・CS） | sns→researcher→customer_success のどこを週次で回すか言語化できている |

---

## フェーズ分解

### Phase 0 — 基盤が「動いている」状態（いま〜2週間）

**ゴール:** Slack・リポ・手渡しの型が一巡し、**何が空いているか**が分かる。

| 順 | タスク | 完了条件（例） |
|----|--------|----------------|
| 0-1 | `meetings/handoffs/当日` の **§0 経営者前提**を毎日 or 週2で埋める | 時間・集中1件が欠けない週が続く |
| 0-2 | 各 `members/*/inbox` を **一斉で15分スキャン**（中身はメモ1行でよい） | 2026-04-23 手渡しの「未スキャン」が解消 |
| 0-3 | **ONE-LINER 依頼**の扱いを決める（各役：返信済み / アーカイブ / 再依頼） | 不要ファイルが減る or 期限付き再依頼が出る |
| 0-4 | Slack ボット（必要なら）**Socket Mode または本番URL**で常時or起動手順が1枚に | 自分が再現できる手順メモ（`projects/slack-bot` README リンクで可） |
| 0-5 | **ONO ポートフォリオ HTML** — 本番URL or クラウドワークス `href` 差し替え | 公開場所が `web` か leader に1行残る |

---

### Phase 1 — 日常運用（毎週回るリズム）

**ゴール:** 朝・夕・週次の**型**に思考が乗る。

| 順 | タスク | 完了条件（例） |
|----|--------|----------------|
| 1-1 | **朝:** `leader/inbox` 一覧 → 本日タスク 3〜7 行（優先度つき） | 経営者と会話する前に空確認できる |
| 1-2 | **夕:** 実施・未完・明日3件 | `handoffs` か Slack のどちらかに数字で残す |
| 1-3 | **週1:** `council-handoff` 1本（`templates` コピー） | 当日ファイルが常に1本ある |
| 1-4 | **月1（任意）:** `personas` と `members/*/CLAUDE` の**ズレ**を1テーマだけ直す | slack-bot かリポのどちらかの整合メモ |
| 1-5 | 営業4役ラベル（探索のみ・発信強化 等）を、**週次で1行** | 会社共通 `CLAUDE` の4役連携と揃う |

---

### Phase 2 — 拡張（営業・セミナー・案件）

**ゴール:** **プロジェクト1本**（例: セミナー用 `projects/…`）が名前から trace できる。

| 順 | タスク | 完了条件（例） |
|----|--------|----------------|
| 2-1 | 次の**セミナー or 営業キャンペーン**の**フォルダ名**を1つ決め、4役に同じ名で `dispatch` | 依頼IDとパスが一致 |
| 2-2 | `sns` × `researcher` × `customer_success` の**今週の1本線**（認知→探索→受け口） | handoffs の「ミニ会議」に1行 |
| 2-3 | X 自動投稿の**正**（iCloud / `x-automation`）を経営者と合意、**RUNBOOK 月次確認** | 方針が文書化されている |

---

### Phase 3 — 最適化（負荷と品質のバランス）

**ゴール:** 週次稼働の上限内で、**捨てる判断**ができる。

| 順 | タスク | 完了条件（例） |
|----|--------|----------------|
| 3-1 | リーダー週次時間の**上限**を経営者と合意 | `CLAUDE.md` の週N時間と一致 |
| 3-2 | `handoffs` の **「今日やらない」**を毎回1行以上 | 何を捨てたか言語化 |
| 3-3 | `.claude` スキルうち**3本だけ**定番化（`CLAUDE.md` にパス1行） | 依頼文が短くて済む |
