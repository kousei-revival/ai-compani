# domain-name-brainstormer

名前を増やすだけでなく、`残すべき候補` を絞る。

## 基本ルール

- 先にコンセプトを決める
- 発音しやすさと覚えやすさを重視する
- TLD まで含めて考える
- 候補は出しっぱなしにしない

## 進め方

### Step 1: 方向性を決める

- 信頼感
- 親しみ
- 速さ
- 高級感

### Step 2: 候補を出す

- 直球型
- 造語型
- 2語結合型

### Step 3: 絞る

- 覚えやすいか
- 誤読しにくいか
- ドメイン候補がありそうか

## 出力フォーマット

```md
## Direction
- ほしい印象

## Candidates
- name / reason

## Shortlist
- 残す候補
```
