# オンボーディング

初回セットアップ。対話を通じてユーザーの `now.md` を作成する。

## 前提

- `now.md` がまだ存在しない（または空）
- `now.example.md` をテンプレートとして参照する

## 進め方

### 全体方針

- 一度に全部聞かない。セクションごとに対話して埋めていく
- 各セクションで2-3の問いかけをして、返答をもとに書く
- 書いたらすぐ見せて「これでいい？」と確認する
- 完璧じゃなくていい。後から `/dump` で磨ける

### Step 0: Perfect Daysとは

いきなり質問に入らず、まずこの場が何なのかを伝える。説明ではなく問いかけで引き込む。

**伝えること：**
- 「perfect day」とは何か — 寝る前に「今日はいい日だったな」と思える日
- ここは何をする場か — 今やるベストなことが見える、考え事に付き合う、振り返って明日を決める。頭の中を一緒に整理する場所
- 今から `now.md`（今の自分の全体像）を一緒に作る。完璧じゃなくていい

ユーザーの反応を待ってからStep 1に進む。

### Step 1: 自分について

```
まず、あなたについて教えてください。
- 名前（呼び方）
- 何をしている人？（仕事、学生、etc.）
- 家族やパートナーは？
- 趣味や好きなこと
```

返答をもとに「自分について」セクションを書く。

### Step 2: 問い

```
今、頭の中にある「問い」はありますか？
答えが出ていなくていいです。ずっと考えていること、気がかりなこと。
```

なければスキップしてOK。後から追加できる。

### Step 3: 原則

```
自分の中で「これだけは守りたい」というルールはありますか？
例: 「朝はスマホを見ない」「週3回は運動する」「寝る前に振り返る」
```

### Step 4: リズム

```
理想の一日の過ごし方を教えてください。
- 何時に起きたい？
- 日中は何をしている？
- 夜は？何時に寝たい？

あと、曜日ごとの決まった予定はありますか？
```

### Step 5: プロジェクト

```
今、並行して走っているものは何がありますか？
仕事、勉強、個人プロジェクト、なんでも。
```

### Step 6: 未決・予定

```
気がかりな未決事項や、直近の予定があれば教えてください。
```

### Step 7: 接し方

```
最後に、私（Claude）にこう接してほしい、という好みはありますか？
例: 「褒めてほしい」「端的に言ってほしい」「考えを引き出す質問をしてほしい」
```

### Step 8: now.md 生成

全セクションが埋まったら `now.md` を生成する。

now.mdの内容をユーザーに見せて確認をとったら：

- 使い方を案内する（`/session`, `/dump`, `/shutdown`）
- そのまま今日の振り返りに入る

## 心がけ

- 質問は軽く。面接にならないようにする
- 「わからない」「まだない」は全然OK。空欄で進める
- 最初から完璧を求めない。使いながら育てるもの
