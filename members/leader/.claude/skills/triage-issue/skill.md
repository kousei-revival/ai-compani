# triage-issue

症状だけで直さず、根本原因を突き止めてから直す。

## 基本ルール

- まず問題文を短く固定する
- すぐ修正案に飛ばない
- `どこで起きるか / なぜ起きるか / 何を直すか` を分ける
- 最後は `TDDでどう直すか` まで落とす

## 進め方

### Step 1: 問題を1文で言う

- 実際に何が起きるか
- 本来どうあるべきか

### Step 2: 関連箇所を探る

- 入口
- 関連フロー
- 近い既存テスト
- 似た実装

### Step 3: 根本原因を言語化する

- 症状ではなく原因を書く
- 依存する条件や再現条件も残す

### Step 4: 修正方針を縦スライスで作る

各サイクルは次の形にする。

- `RED`: 壊れた振る舞いを表すテスト
- `GREEN`: そのテストを通す最小修正

### Step 5: issue化するか残す

- GitHubがあるなら issue
- なければ `projects/` に調査メモや修正計画を残す

## 出力フォーマット

```md
## Problem
- 実際の挙動
- 期待挙動

## Root Cause
- 原因

## TDD Fix Plan
1. RED:
   GREEN:
2. RED:
   GREEN:
```
