# frontend-design

ただ動くUIではなく、`目的に合った見た目` を作る。

## 基本ルール

- 先に用途と受け手を決める
- 美しさより `意図のある一貫性` を優先する
- ありがちな AI デザインを避ける
- タイポ、色、余白、視線誘導をセットで考える

## 進め方

### Step 1: 方向性を決める

- 誰向けか
- 何を感じてほしいか
- どこが印象に残るべきか

### Step 2: 見た目の柱を決める

- 色
- 書体
- 余白
- 角丸や線の強さ
- 動き

### Step 3: レイアウトを作る

- 主役の要素
- 補助情報
- 行動導線

### Step 4: 仕上げる

- 情報量の強弱
- コントラスト
- 画面ごとの一貫性

## 出力フォーマット

```md
## Design Direction
- トーン
- 想定ユーザー

## Visual System
- 色
- 書体
- 余白

## Key UI Decisions
- 決定1
- 決定2
```
