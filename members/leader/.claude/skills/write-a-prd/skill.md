# write-a-prd

ふわっとした案を、実装前に読めるPRDへ変える。

## まずやること

- `now.md` と関連する `projects/` を読む
- 関連コードや既存仕様があるなら確認する
- すでに決まっていることを先に回収する

## 進め方

### Step 1: 問題を明確にする

次を短く言語化する。

- 何が問題か
- 誰が困っているか
- 何が変われば成功か

### Step 2: ユーザーに深掘りする

足りない部分だけ質問する。

- 使う場面
- 例外ケース
- 制約
- 今回やらないこと

### Step 3: 実装観点を整理する

必要なら次を言葉にする。

- 主要モジュール
- データの流れ
- 外部サービス境界
- どこを重点的にテストするか

### Step 4: PRDにまとめる

テンプレートは [template.md](template.md) を使う。

- GitHub issue が必要なら issue 化する
- そうでなければ `projects/` に残す

## このプロジェクト向けのルール

- まず `projects/` に読める文章で残す
- ファイル名は `projects/<テーマ>-prd.md` を基本にする
- `now.md` に載せる価値があるなら索引も更新する

## 出力の確認

- 問題と解決策がズレていないか
- スコープ外が明記されているか
- 実装判断に使える粒度になっているか
