# tdd

まとめてテストを書かず、1本ずつ `RED -> GREEN -> REFACTOR` を回す。

## 基本ルール

- テストは `実装の中身` ではなく `外から見える振る舞い` を見る
- 一度に全部のテストを書かない
- 1サイクルで扱うのは `1つの振る舞い`
- GREENでは `今のテストが通る最小の変更` だけ入れる
- REFACTORは `GREENのあと` にだけやる

## 進め方

### Step 1: 何をテストするか決める

- どの振る舞いがいちばん重要か
- 公開インターフェースは何か
- 今回はどこまで見るか

### Step 2: 失敗するテストを1本書く

- まず失敗することを確認する
- 実装詳細に寄りすぎない

### Step 3: 最小実装で通す

- 次のテストのためのコードは先回りしない
- 今の1本を通すところまでに絞る

### Step 4: もう1本必要か判断する

- 新しい振る舞いなら次のREDへ
- 同じ振る舞いの整理ならREFACTORへ

### Step 5: リファクタする

- 重複除去
- 境界の整理
- 深いモジュール化

## 出力フォーマット

```md
## 今回の振る舞い
- 何を確認するか

## RED
- 書くテスト

## GREEN
- 最小変更

## REFACTOR
- 整理ポイント
```
