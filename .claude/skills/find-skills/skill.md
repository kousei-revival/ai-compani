# find-skills

新しいSkillを書く前に、まず既存のものを探す。

## 基本ルール

- いきなり新規作成に行かない
- `何をしたいか` と `いつ発火してほしいか` を先に1行で言語化する
- 候補は多く出しすぎず、まず `3件以内` に絞る
- `そのまま使える / 少し直せば使える / 参考だけにする` の3段階で判定する

## 調査順

1. `Anthropic公式`
2. `Matt Pocock`
3. `SkillsMP` などの公開マーケットプレイス
4. ユーザーが貼ったGitHubリンクや既存リポジトリ

詳しい探し先は [sources.md](sources.md) を見る。

## 調査の進め方

### Step 1: 探すテーマを固定する

次の3つを短く決める。

- このSkillで何をできるようにしたいか
- どんな場面で呼ばれたいか
- このプロジェクト向けに何を足したいか

## Step 2: 候補を集める

各ソースから、近いものだけを拾う。

- 名前
- リンク
- 何ができるか
- そのまま使えそうか

## Step 3: 再利用方針を決める

- `7割以上そのまま使える` -> 流用
- `骨格は使えるが言い回しや手順を変えたい` -> フォークして自分用化
- `近いものがない` -> 新規作成

## Step 4: 自分用の差分を言語化する

特に次を明確にする。

- 日本語で書くか
- 初心者向けにどこまで噛み砕くか
- 自分の実務フローに合わせて何を先にやるか
- 出力テンプレートやチェックリストを付けるか

## 出力フォーマット

```md
## 候補
- `skill-name`: 何ができるか。流用 / フォーク / 参考のみ

## 採用方針
- 今回は `流用 / フォーク / 新規作成`

## 自分用に変える点
- 変更点1
- 変更点2
- 変更点3
```
