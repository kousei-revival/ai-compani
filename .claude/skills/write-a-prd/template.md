# PRD Template

```md
# [企画名]

## Problem
- 何が問題か
- 誰が困っているか
- 今のままだと何が起きるか

## Solution
- 何を作るか
- どう変えるか
- 成功した状態

## User Stories
1. [利用者] として、[したいこと] ができる
2. [利用者] として、[したいこと] ができる

## Implementation Decisions
- 大きなモジュール
- データの流れ
- 外部サービスや依存
- 先に決めるべきこと

## Testing Decisions
- どこを重点的にテストするか
- 壊れると痛い動作

## Out of Scope
- 今回やらないこと

## Further Notes
- 補足
```

## メモ

- まずは読んで判断できる文章にする
- 実装詳細を書きすぎない
- ただし判断材料になる制約は落とさない
