# write-a-skill

このプロジェクトでは、Skillは `.claude/skills/<skill-name>/skill.md` に置く。

## 先に決めること

次の4つを曖昧にしない。

1. このSkillで何をできるようにするか
2. どんな言葉や状況で発火してほしいか
3. 何を出力できれば成功か
4. 既存Skillを流用するのか、新規で書くのか

## 作り方

### Step 1: 目的を1行にする

まず `このSkillは何のためのものか` を1行で書く。

### Step 2: 名前と説明文を決める

- `name` は短く、役割がわかるものにする
- `description` には `何をするか` と `いつ使うか` を両方入れる
- 発火しやすいように、ユーザーが言いそうな語を入れる

### Step 3: 本文は最小構成で書く

基本はこの順で書く。

1. 何のSkillか
2. 最初にやる判断
3. 実行ステップ
4. 出力フォーマット
5. 必要なら追加資料へのリンク

テンプレートは [template.md](template.md) を使う。

### Step 4: このプロジェクト向けに寄せる

自分用化するときは、特に次を足す。

- 日本語で迷わず使える言い回し
- 自分の仕事順に合ったステップ順
- よく詰まる点の先回り
- 必要ならテンプレートやチェックリスト

### Step 5: 長くなりすぎたら分ける

- `skill.md` は短く保つ
- 詳細は別ファイルへ逃がす
- 参照先は1段だけにする

## チェック

- `description` に発火条件が入っているか
- 説明だけでなく、手順が書かれているか
- 抽象論だけで終わっていないか
- 未来の自分が読んで迷わないか
