# DOCX / XLSX Playbook

## DOCX

### まずやる判断
- 新規作成か
- 既存文書の修正か
- 変更履歴やコメントが必要か

### 編集の基本
1. 既存文書か新規文書かを分ける
2. 既存文書なら、小さい差分で直す
3. 変更履歴やコメントの要否を先に決める
4. 大きな書き換えより、最小単位で安全に直す

### DOCXで意識すること
- 見た目よりXMLの整合性が大事
- tracked changes や comments は位置関係を壊しやすい
- 「全部まとめて修正」は事故のもと

## XLSX

### まずやる判断
- 分析中心か
- 既存シート修正か
- 新規レポート作成か

### 編集の基本
1. 値を置くのではなく、数式で持たせられるか考える
2. 既存テンプレがあるなら、そのルールを崩さない
3. 保存したら再計算する
4. 数式エラーがないか確認する

### XLSXで意識すること
- Pythonの計算結果をベタ書きしすぎない
- `#REF!`、`#DIV/0!`、`#VALUE!` を見逃さない
- 再計算しないまま終わらせない

## 共通の締め

`docx` と `xlsx` は、どちらも「出力できた」だけでは不十分。

- `docx`: 壊さず直せているか
- `xlsx`: 計算が生きているか

この確認まで終えてはじめて完了とする。
