# PDF / PPTX Playbook

## PDF

### まずやる判断
- 読み取りか
- 加工か
- フォーム入力か
- スキャンPDFか

### 読み取りの基本
1. まず本文が抽出できるか確認する
2. 表があるなら、本文抽出と表抽出を分けて考える
3. テキストが取れないなら、画像ベースPDFの可能性を考える

### フォーム入力の基本
1. fillable fields があるか先に判定する
2. あるなら field 情報を取り出して値を埋める
3. ないなら annotation で埋める流れに切り替える
4. 出力後に必ず見た目を確認する

### PDFで意識すること
- 本文抽出と表抽出は別タスクとして扱う
- OCRが必要かを早めに見極める
- フォームは `fillable` と `non-fillable` で流れが大きく変わる

## PPTX

### まずやる判断
- 既存デッキを読むのか
- 既存デッキを直すのか
- 新規で作るのか

### 読み取りの基本
1. 全体のスライド構成を把握する
2. 本文テキストを抜き出して中身を見る
3. 各スライドの役割を分類する

### 編集の基本
1. 先にスライド構成を決める
2. 構造変更を終えてから本文を直す
3. 1枚ごとに直し、最後に全体で整合性を見る
4. PDF化や画像化で見た目を検証する

### 新規作成の基本
1. まず3枚程度の最小デッキで作る
2. 配色、余白、タイトルサイズのルールを先に固定する
3. 1枚1メッセージで構成する

### PPTXで意識すること
- テンプレ編集は「構造」と「文言」を分離する
- bullet や色指定などの地雷を先に避ける
- QAなしで完了扱いにしない
