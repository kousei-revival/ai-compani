# 役職者会議 → 経営者への手渡し

## 何のためか

毎回、**leader が仮想の役職者会議**を開き、各メンバーの状況を踏まえて **1 枚の Markdown** にまとめ、**経営者（あなた）**へ渡す。  
ToDo のマトリクス仕分けや「今日 7 件」も、**このファイルを正**にすれば再現できる。

## 置き場

| パス | 内容 |
|------|------|
| `meetings/handoffs/YYYY-MM-DD.md` | **その日の手渡し**（会議の議事と結論） |
| `templates/council-handoff-template.md` | **雛形**（コピーして日付ファイルを作る） |

## 回し方（leader の作業）

1. 雛形を `handoffs/今日の日付.md` にコピーする。  
2. **経営者からの前提**（使える時間・いまの時刻・集中したい 1 件）を埋める（inbox にあれば転記）。  
3. **各 `members/*/inbox/` と直近の `projects/`** を見て、ミニ会議の欄を埋める（活動がなければ「なし」）。  
4. **マトリクス・今日 7 件・今日やらない**を leader が書く。  
5. **経営者確認事項**だけを最後に残す。  
6. 経営者は **`meetings/handoffs/` の今日のファイル**だけ見ればよい。

`Cursor` では `cd ~/ai-company/members/leader` で起動し、上の手順をそのままプロンプトに書いてもよい。

## 経営者側の一言プロンプト例

```text
leader として今日の council-handoff を出して。
前提：作業6時間、今は（時刻）、集中は（1件）。
ToDo は以下：…
```

雛形に沿った **`meetings/handoffs/YYYY-MM-DD.md` の全文**を出してもらい、そのまま保存する。
